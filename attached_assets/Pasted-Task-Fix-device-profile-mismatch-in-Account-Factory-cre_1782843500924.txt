Task: Fix device-profile mismatch in Account Factory credential pool

The bug

The Account Factory is pairing operator-supplied Telegram api_ids (registered on my.telegram.org as Android apps) with Desktop Windows fingerprint strings. Telegram's anti-automation treats this platform mismatch as an instant bot signal and returns SentCodeTypeApp for every number — even fresh, never-used ones.

Evidence from the debug log:

TELETHON_INIT: { "api_id": 37989276, "platform": "desktop", "device_model": "PC 64bit", "system_version": "Windows 11", "app_version": "5.9.5" }
api_id 37989276 was registered on my.telegram.org as Android. It should be paired with an Android device fingerprint, not Desktop.

File to edit

account_factory.py (repo root)

Background context

Replit previously applied a credential-loader change. The credential loader exists but its Android device profiles are NOT being used — somewhere the code is still hardcoding ("PC 64bit", "Windows 11", "5.9.5", "desktop") for operator credentials. Find that hardcoded fallback and remove it.

Both operator api_ids (TELETHON_API_ID_1=30533575 and TELETHON_API_ID_2=37989276) were registered on my.telegram.org as Android apps. Both must use Android device profiles.
